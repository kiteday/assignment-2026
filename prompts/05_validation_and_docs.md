# Prompts - 05. 검증 및 문서 정리

## 질문 1: 수동 테스트 시나리오

### 프롬프트
```
핵심 실패 케이스(정원 초과, 학점 초과, 시간 충돌, 중복 신청)를
curl로 검증하는 방법을 알려줘.
```

### Codex의 답변
각 케이스별로 최소 1개씩 재현 가능한 커맨드를 제공하고,
응답 코드(`CAPACITY_EXCEEDED`, `CREDIT_EXCEEDED`, `TIME_CONFLICT`, `ALREADY_ENROLLED`)
기준으로 성공/실패를 판별하세요.

---

## 질문 2: 문서 반영 범위

### 프롬프트
```
README, REQUIREMENTS, API 문서에 무엇을 기록해야 하는지 정리해줘.
```

### Codex의 답변
- README: 실행 방법, 포트 정보, 테스트 요약
- REQUIREMENTS: 요구사항 분석, 설계 결정, 동시성 전략
- API: 모든 엔드포인트 요청/응답 + 에러 케이스

---

## 질문 3: 최종 점검표 구성

### 프롬프트
```
제출 직전 체크리스트 형식으로 점검표를 만들고 싶다.
필수 기능과 문서 항목을 포함하도록 정리해줘.
```

### Codex의 답변
- `/health` 200 OK
- 학생/강좌/교수 조회 동작
- 수강신청/취소/시간표 동작
- 정원/학점/시간충돌/중복 신청 에러 확인
- 동시성 테스트 통과

---

## 정리

검증 및 문서 정리 단계에서 다음을 결정했습니다:

1. **수동 테스트 케이스**: 핵심 실패 케이스별 curl 시나리오 구성
2. **문서 반영**: README/REQUIREMENTS/API 문서에 테스트 요약 및 실행 정보 추가
3. **체크리스트**: 최종 점검표를 REQUIREMENTS에 포함
